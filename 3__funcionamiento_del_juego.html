3 - Funcionamiento del juego

Las modificaciones que se proponen en esta página son de carácter voluntario

Creamos las variables puntos y vidas

Para mejorar el funcionamiento del juego, vamos a añadir dos variables fundamentales en la ejecución de cualquier juego: los puntos y las vidas.

En primer lugar creamos la variable vida (lives) con valor 3 (o el que prefieras) y la variable puntuación (score) con valor 0.

Estas variables se colocan en la parte superior del código, por ejemplo tras la definición del título y dimensiones de la ventana.

ver código

score = 0
lives = 3

Mostramos esta información en la pantalla. Para ello, añadiremos el siguiente código dentro de la función inicial draw:

  • Dibujaremos en la pantalla el texto "Score:" y el valor de la variable de puntos. Lo mostraremos en la posición 10,10 (esquina superior izquierda) y de color blanco.
  • Dibujaremos en la pantalla el texto "Lives:" y el valor de la variable de vidas. Lo mostraremos en la posición 10, ancho de ventana-10 (esquina superior derecha) y de color blanco.
  • Si las vidas es menor o igual a 0
    • Dibujaremos en la pantalla el texto "GAME OVER". Lo mostraremos centrado en la ventana (width/2 , height/2) y de color rojo.

ver código

def draw():
    screen.fill("black")
    screen.draw.filled_rect(Rect((0, 0), (WIDTH, 50)), "green")
    player.draw()
    for vehicle in vehicles:
        vehicle.draw()
    screen.draw.text(f"Score: {score}", (10, 10), color="white")
    screen.draw.text(f"Lives: {lives}", (WIDTH - 100, 10), color="white")
    if lives <= 0:
        screen.draw.text("GAME OVER", center=(WIDTH // 2, HEIGHT // 2), fontsize=50, color="red")

Añadimos puntos y quitamos vidas

Añadimos el código para contabilizar un punto cada vez que el jugador alcance la zona segura.

Para ello, incluimos el incremento de la variable puntuación (score) en la función update_player.

Además debemos incluir la referencia global a la variable de puntuación (score).

ver código

def update_player():
	global score
    player_speed = 3
    if keyboard.left and player.x > 0:
        player.x = player.x - player_speed
        player.image = 'frog_left'
    if keyboard.right and player.x < WIDTH:
        player.x = player.x + player_speed
        player.image = 'frog_right'
    if keyboard.up and player.y > 0:
        player.y = player.y - player_speed
        player.image = 'frog_up'
    if keyboard.down and player.y < HEIGHT:
        player.y = player.y + player_speed
        player.image = 'frog_down'
    if player.y < 50:
        player.pos = (WIDTH // 2, HEIGHT - 50)
        score = score + 1

Por el contrario, la pérdida de vidas ocurrirá al colisionar el jugador con un vehículo.

Para ello añadiremos en la función update_vehicles, el decremento de la variable vidas (lives) al colisionar con un vehículo.

Además debemos incluir la referencia global a la variable de vidas (lives).

ver código

def update_vehicles():
    global lives
    vehicle_speed = 3
    for vehicle in vehicles:
        vehicle.x = vehicle.x + vehicle_speed
        if vehicle.x > WIDTH:
            vehicle.x = -50
    for vehicle in vehicles:
        if player.colliderect(vehicle):
            player.pos = (WIDTH // 2, HEIGHT - 50)
            lives = lives - 1

Por último, aunque hemos mostrado el mensaje GAME OVER, habrás observado que el juego no se detiene. Esto se debe a que a pesar del mensaje, la función update continúa su ejecución.

Para forzar la detención del juego, debemos añadir el siguiente código dentro de la función update, al principio antes del resto de código.

  • Si las vidas son igual o menor a 0
    • añadir la instrucción return.

Esta instrucción obliga a finalizar la función, sin llegar a ejecutar el resto de código update_player y update_vehicles. De esta forma conseguimos el efecto de detención del juego

ver código

def update():
    if lives <= 0:
        return  
    update_player()
    update_vehicles()

Cambiar el sentido de algunos vehiculos

Vamos a hacer una modificación voluntaria al juego. Como van a ser muchos cambios, te recomiendo que hagas copia del juego que ya has realizado y funciona correctamente.

En esta ocasión vamos a cambiar el funcionamiento del juego para que algunos vehiculos se muevan de izquierda a derecha (como hasta ahora), pero otros lo hagan en sentido contrario de derecha a izquierda.

En primer lugar vamos a cambiar la creación de los vehículos, en lugar de tener una lista vehicles, vamos a tener dos listas vehicles_left y vehicles_right.

Al crear los vehículos, calculamos una variable pos aleatoria con valor 0 o 1.

Si el valor pos es 0, el vehículo irá de izquierda a derecha y para ello lo añadimos a la lista vehicles_right.

En caso contrario, si el valor es 1, el vehículo irá de derecha a izquierda y para ello lo añadimos a la lista vehicles_left. En este caso, vamos a utilizar una imagen invertida del coche "car_inv".

ver código

Vamos a cambiar este código

vehicles = []
y = 50
num_vehicles = 8
for i in range(num_vehicles):
    x = random.randint(0, WIDTH)
    y = y + 40
    car = Actor("car", (x, y))
    vehicles.append(car)

Por este otro

vehicles_left = []
vehicles_right = []
y = 50
num_vehicles = 8
for i in range(num_vehicles):
    x = random.randint(0, WIDTH)
    y = y + 40
    pos = random.randint(0,1)
    if pos==0:
        vehicles_right.append(Actor("car", (x, y)))
    else:
        vehicles_left.append(Actor("car_inv", (x, y)))

Para dibujar correctamente los vehículos, vamos a modificar el código de la función draw.

En esta función, vamos a cambiar el bucle que recorre la lista vehicles, por una suma de las listas vehicles_left + vehicles_right.

Las listas en Python se pueden sumar uniendo sus elementos.

ver código

Cambiamos el siguiente código

def draw():
    screen.fill("black")
    screen.draw.filled_rect(Rect((0, 0), (WIDTH, 50)), "green")
    player.draw()
    for vehicle in vehicles:
        vehicle.draw()

Por este otro

def draw():
    screen.fill("black")
    screen.draw.filled_rect(Rect((0, 0), (WIDTH, 50)), "green")
    player.draw()
    for vehicle in (vehicles_left + vehicles_right):
        vehicle.draw()

El siguiente paso es gestionar el movimiento de los vehículos de una lista en un sentido y los vehículos de la otra lista en sentido contrario.

Para ello, dentro de la función update_vehicles, cambiaremos el primer bucle que recorre la lista vehicles. El segundo bucle, por ahora no lo modificaremos.

  • Cambiamos el nombre de la lista vehicles por vehicles_right.
    • El resto del bucle permanece sin cambio.
  • Duplicamos todo el bucle creando una nueva copia igual a la anterior, y cambiamos el nombre de la lista de vehicles_right a vehicles_left
    • En esta ocasión la posición x del vehículo se resta a la velocidad del vehículo (ya que vamos en sentido contrario).
    • Comprobamos si el vehículo supera el margen izquierdo, es decir x<0.
      • En ese caso, posicionamos el vehículo por fuera del margen derecho, es decir, el tamaño de la ventana (width) + 50.

ver código

Vamos a cambiar este código

def update_vehicles():
    global lives
    vehicle_speed = 3

    for vehicle in vehicles:
        vehicle.x = vehicle.x + vehicle_speed
        if vehicle.x > WIDTH:
            vehicle.x = -50

Por este otro

def update_vehicles():
    global lives
    vehicle_speed = 3

    for vehicle in vehicles_right:
        vehicle.x = vehicle.x + vehicle_speed
        if vehicle.x > WIDTH:
            vehicle.x = -50

Por último, dentro de esta misma función update_vehicles, vamos a modificar el bucle que controla las colisiones con el jugador.

Modificamos el último bucle (for) cambiando la lista vehicles por la suma de vehicles_left + vehicles_right.

ver código

Cambiamos este código

def update_vehicles():
